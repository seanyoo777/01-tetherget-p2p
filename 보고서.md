## 요약

P2P는 **@tetherget/core 정책**, **서버 게이트**, **프론트 브리지·고액 모달**, **KYC 문서 서버 암호 저장**까지 골격이 있다. **최근 구현**: 주문에 **`PATCH /api/p2p/orders/:id/onchain-escrow`** 로 `metadata_json.onchain_escrow_id` 저장·수정, 진행 주문 카드에서 **`P2pOnchainEscrowBlock`** 으로 체인 조회 + buyer **`confirmReceipt`**, 고액 mark-paid 시 **`p2p_high_value_ack_logs`** 적재, **`risk_ip_denylist`** 를 게이트 IP 블랙리스트에 합산, KYC 스위치 **45초 폴링**으로 DB와 localStorage 동기. **남은 과제**: createEscrow **TX 자동 감지·ID 자동 기록(인덱서)**, **실 푸시 FCM·수신 로그**, **레퍼럴 정산 DB**, **League 런타임**. Exchange·League는 core 공유로 이후 확장.

## 7단계 목표 대비

| 목표 | 상태 |
|------|------|
| 온체인 escrow ID ↔ 주문 매핑 + confirmReceipt 연동 | **진행** — API로 메타 병합·이벤트 `onchain_escrow_linked`; UI에서 Funded + buyer 지갑 일치 시 `confirmReceipt`. **미완**: 블록 인덱서로 TX 후 자동 기록. |
| KYC Vault + 서버 암호화 저장 | **달성** — `kyc_documents` 등 암호화 저장 경로 존재. 운영 키·회전은 운영 과제. |
| 고액 푸시 + 모달 + 서버 로그 | **부분** — 모달→API→`gateBuyerMarkPaid` + **고액 시** `p2p_high_value_ack_logs` (추정 KRW·푸시/모달 플래그). **미완**: FCM 등 실제 푸시 발송·수신 로그. |
| 관리자 KYC 스위치 DB + 실시간 동기화 | **부분** — `platform_settings` + 로그인 pull + **45초 폴링** + `CustomEvent`. (SSE/WS는 선택) |
| Security 자동 차단 서버 실행 | **부분** — `risk_ip_denylist` 가 IP 게이트에 **합산**됨. 지문·다중계정·자동차단 DB는 추가 여지. |
| 레퍼럴 풀·분배 (수수료→풀, 무한 단계 bps) 실제 적용 | **코어 완비 / 정산 연동 과제** — `referral/*` 풀·분배·감사. 실거래 수수료 징수→DB는 별도. |
| League 10분 시세 스냅샷 + 정책 | **코어 준비** — 상수·`isLeaguePriceSnapshotStale`. 스냅샷 저장·매치 엔진 미구현. |

## 앞으로 우선순위 (짧게)

1. createEscrow **TX 리ceipt / 이벤트**로 `onchain_escrow_id` **자동** 기록
2. **푸시** 발송 ID·수신 확인 테이블 + 모달 플래그와 조인
3. 레퍼럴 수수료 확정 시 감사 로그 append-only 저장
4. KYC 스위치 **SSE** (폴링 대체·보완)
